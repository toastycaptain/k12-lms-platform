# PYP Vertical Slice

## Golden path

1. Create PYP unit from `/ib/pyp/units/new`.
2. Land directly in `/ib/pyp/units/[documentId]`.
3. Edit the live PYP unit record and related weekly-flow/family-facing content.
4. Capture and validate evidence tied to the active unit.
5. Request reflection and draft a family story/window.
6. Move the item through review and publishing.
7. Guardian and student surfaces consume the published output.

## In scope

- live PYP unit creation
- live unit studio
- weekly-flow route
- evidence linkage
- family story/window linkage
- coordinator review return/approve loop
- guardian/student output for published slice content

## Out of scope for this slice

- full MYP vertical slice
- full DP vertical slice
- full exhibition programme release beyond the hook/seed record

## Release flag

- `ib_pyp_vertical_slice_v1`

## Definition of done

- no demo IDs in the PYP route journey
- PYP create/open/edit/review/publish flow runs on live records
- guardian/student output reflects only published, permission-safe content
